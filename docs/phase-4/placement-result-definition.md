# Phase 4 — Placement Result Semantics

## Purpose

This document defines the exact meaning of every output produced by the Phase 4 placement test result. It ensures that backend services, admin consumers, and Flutter display all interpret placement outputs consistently.

All outputs defined here are generated exclusively by the backend. Flutter must not calculate, estimate, or modify any of these values locally.

---

## Result Outputs Overview

| Output | Meaning | Generated By |
|---|---|---|
| Estimated Level | Learner's starting CEFR level | Backend scoring service |
| Skill Map | Per-skill signal from placement answers | Backend result service |
| Weakness Map | Skills where learner showed lowest signal | Backend result service |
| Initial Learning Path | Ordered skill sequence to start learning | Backend path service |

---

## 1. Estimated Level

### Definition

The estimated level is the backend's determination of the learner's approximate CEFR level based on placement test performance. It is an estimate only — not a certified assessment result.

### Valid Values

| Value | Meaning |
|---|---|
| A1 | Beginner — very limited English ability |
| A2 | Elementary — basic English ability |
| B1 | Intermediate — conversational English ability |

Phase 4 covers A1, A2, and B1 only. B2, C1, and C2 are out of scope for Phase 4.

### How It Is Derived

1. Backend scores each section using predefined section weights.
2. Backend sums weighted scores into a total placement score (0-100).
3. Backend maps the total score to a CEFR level using threshold rules defined in P4-030.
4. Backend writes the estimated level to the placement_results table.

### What It Is Not

- It is not a certified CEFR score.
- It is not calculated by Flutter.
- It is not produced by the AIM Engine runtime.
- It does not update after the placement attempt is complete.

### Flutter Display Rule

Flutter displays the estimated level label (e.g. "A2") returned from the result API. Flutter must not recalculate or override this value.

---

## 2. Skill Map

### Definition

The skill map is a per-skill signal derived from the learner's placement answers. It shows which skills the learner demonstrated during the placement test.

### Signal Values

| Signal | Meaning |
|---|---|
| strong | Learner answered most questions for this skill correctly |
| developing | Learner answered some questions for this skill correctly |
| emerging | Learner answered few or no questions for this skill correctly |

### How It Is Derived

1. Backend reads placement_question_skills to find which skills each question is linked to.
2. Backend reads placement_answers to find which answers were correct.
3. Backend computes a correctness ratio per skill.
4. Backend maps the ratio to a signal value using rules defined in P4-032.
5. Backend writes the skill map to the placement_results table as a structured record.

### What It Is Not

- It is not a mastery score — it is a placement signal only.
- It is not calculated by Flutter.
- It does not reflect lesson performance (that is Phase 5+).
- It is not produced by the AIM Engine runtime.

### Flutter Display Rule

Flutter may display skill signal labels (e.g. "Grammar: developing") from the skillSummary field in the result API. Flutter must not recalculate signals or derive mastery locally.

---

## 3. Weakness Map

### Definition

The weakness map identifies the subset of skills from the skill map where the learner showed the lowest signal. It is used to prioritize which skills appear first in the initial learning path.

### How It Is Derived

1. Backend reads the skill map produced in step 2 above.
2. Backend selects skills with signal "emerging" or the lowest-ratio "developing" skills.
3. Backend orders them by correctness ratio ascending (lowest first).
4. Backend writes the weakness map to the placement_results table.

### What It Is Not

- It is not a permanent weakness record — it reflects placement only.
- It is not calculated by Flutter.
- It does not update after placement is complete.
- It is not produced by the AIM Engine runtime.

### Flutter Display Rule

Flutter does not display the weakness map directly. The weakness map is used by the backend to generate the initial learning path. Flutter only displays the initial path output.

---

## 4. Initial Learning Path

### Definition

The initial learning path is an ordered sequence of skills that the learner should study first, based on the weakness map and skill map from the placement result.

### How It Is Derived

1. Backend reads the weakness map.
2. Backend reads the skill map for context on developing skills.
3. Backend applies initial path rules defined in P4-034 to produce an ordered skill list.
4. Backend writes the initial path to the database as defined in P4-024.

### What It Is Not

- It is not an adaptive path — it is a static starting sequence only.
- It is not calculated by Flutter.
- It does not include lesson content (that is Phase 5+).
- It is not produced by the AIM Engine runtime.
- It is not a recommendation engine output.

### Flutter Display Rule

Flutter may display the initial path as a list of skill names in order. Flutter must not reorder or filter the path locally. The backend is the sole authority on path order.

## Result Lifecycle

Placement Attempt Completed
        |
        v
Backend Scoring Service
  - Computes weighted section scores
  - Derives total placement score
  - Maps score to estimated level
        |
        v
Backend Result Service
  - Builds skill map from question-skill links and answer correctness
  - Builds weakness map from skill map
  - Writes placement_results record
        |
        v
Backend Path Service
  - Reads weakness map and skill map
  - Applies initial path rules
  - Writes initial learning path record
        |
        v
Result Available via GET /placement/attempts/:id/result

---

## What Flutter May Display

| Field | Allowed Display |
|---|---|
| estimatedLevel | Show as learner's starting level label |
| skillSummary[].skillName | Show skill name |
| skillSummary[].signal | Show signal label (strong / developing / emerging) |
| initialPathReady | Show whether the path is ready |

## What Flutter Must Never Display or Calculate

| Forbidden | Reason |
|---|---|
| Raw section scores | Internal backend value |
| Correctness ratios | Internal backend value |
| Score thresholds | Internal backend configuration |
| Weakness rank order | Internal backend value |
| Recalculated level | Backend authority only |
| Recalculated skill signals | Backend authority only |
