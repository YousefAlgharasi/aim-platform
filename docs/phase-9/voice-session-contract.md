## Purpose

This document defines the exact shape of a Voice Session as it flows
between the Voice API Endpoints, Voice Session Orchestration, and Voice
Persistence layers described in `docs/phase-9/voice-architecture.md`.
It keeps the session object predictable and consistent across every
backend component that creates, reads, or extends a voice session,
before any of those components are implemented. It is the internal
counterpart to the client-facing shapes already fixed in
`docs/phase-9/voice-api-map.md`.

---

## Session Contract

A Voice Session, as held internally by the backend, has this shape:

```jsonc
{
  "sessionId": "string — opaque session identifier",
  "studentId": "string — owning student, set server-side only",
  "contextRef": "string — lesson/topic identifier used to seed Context Builder",
  "status": "string — one of: active | ended",
  "createdAt": "string — ISO 8601 timestamp",
  "updatedAt": "string — ISO 8601 timestamp"
}
```

- **`sessionId`** is generated by the backend on creation; it is never
  supplied by the client.
- **`studentId`** is set from the authenticated request's student
  identity at creation time; it is never accepted as a request field
  (see `docs/phase-9/no-aim-authority-change-rule.md` and
  `docs/phase-9/voice-api-map.md` §4 — "ownership is set server-side,
  never from a client-supplied student id").
- **`contextRef`** is the same lesson/topic identifier accepted by
  `POST /ai-teacher/voice/sessions` (per `docs/phase-9/voice-api-map.md`
  §4); it is passed through unchanged to the Context Builder for every
  turn in the session.
- **`status`** tracks whether the session is still accepting turns
  (`active`) or has been closed (`ended`); it is a session lifecycle
  flag, not a learning-decision value.
- **`createdAt`** / **`updatedAt`** are normal timestamps for ordering
  and lifecycle tracking, not learning-decision fields.

No other top-level field is part of the contract. In particular, the
following are never part of a Voice Session object:

| Excluded field | Reason |
|---|---|
| `masteryScore`, `level`, `weakness`, `difficulty`, `recommendation`, `reviewSchedule`, or any similarly named field | A voice session is a conversation container, not a learning-decision record (`docs/phase-9/no-aim-authority-change-rule.md`). |
| `sttProvider`, `ttsProvider`, `aiProvider`, `apiKey`, or any provider/credential-shaped field | Provider identity and credentials stay backend-internal; never attached to the session object exposed beyond the gateways (`docs/phase-9/no-client-provider-rule.md`). |
| `rawAudioPath`, `rawAudioBytes` | The session references turns via `audioRef` (see Turn Contract below); it never embeds a raw audio file path or byte payload. |

## Turn Contract (within a Session)

Each voice turn belonging to a session has this shape internally,
matching the client-facing shape already fixed in
`docs/phase-9/voice-api-map.md` §1–2:

```jsonc
{
  "turnId": "string — opaque turn identifier",
  "sessionId": "string — owning session",
  "transcript": "string | null",
  "reply": "string | null",
  "audioRef": "string | null — opaque reference, resolved via audio stream endpoint",
  "status": "string — one of: pending | transcribed | replied | synthesized | failed",
  "createdAt": "string — ISO 8601 timestamp"
}
```

- **`turnId`** is generated by the backend; never client-supplied.
- **`transcript`**, **`reply`**, **`audioRef`** are populated
  progressively as the turn moves through the lifecycle described in
  `docs/phase-9/voice-request-lifecycle.md`; each may be `null` until
  its producing step completes.
- **`status`** mirrors the lifecycle stage (matches
  `VoiceTurnStatus` in
  `services/backend-api/src/features/voice-teacher/voice-teacher.constants.ts`)
  and is for internal/observability use; the client-facing shape in
  `docs/phase-9/voice-api-map.md` does not require exposing `status`
  directly, only the final `transcript`/`reply`/`audioRef`/`createdAt`.
- **`audioRef`** is always an opaque reference, never a raw file path,
  provider URL, or filesystem location.

---

## Predictability Rules

- `sessionId` and `turnId` are always present and non-null once a
  session/turn exists; only the in-progress turn fields (`transcript`,
  `reply`, `audioRef`) may be `null` while the turn is still being
  processed.
- `status` values are restricted to the enumerated sets above; no
  free-form status string is introduced.
- All timestamps are ISO 8601 strings, used for ordering and display,
  not for learning-decision logic.

---

## Where This Contract Applies

- **Voice Session Orchestration (Group B/E):** creates and updates
  session/turn records using exactly this shape internally.
- **Voice Persistence (Group E):** persists session/turn records using
  this shape, writing only to voice-session/voice-turn tables, never to
  an AIM Engine-owned table or column.
- **Voice API Endpoints (Group F):** map this internal shape to the
  client-facing response shapes fixed in
  `docs/phase-9/voice-api-map.md`; internal-only fields (`studentId`,
  `status`, `updatedAt`) are not re-exposed to the client beyond what
  the API map already declares.

---

## Validation

- The Voice Session and Voice Turn shapes are defined as predictable,
  bounded objects before implementation begins.
- The contract excludes any learning-decision field, provider/credential
  field, or raw audio path.
- AI Teacher Voice Mode does not replace AIM Engine authority via this
  contract.
- Flutter does not call an STT, TTS, or AI provider, or the AIM Engine,
  directly.
- No secrets are referenced or committed in this document.
