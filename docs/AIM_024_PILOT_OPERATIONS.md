# AIM-024 Pilot Operations

This runbook turns the AIM-023 readiness plan into a day-by-day operating process for the two-week web pilot.

## Operator Routine

1. Confirm each learner can sign in before the scheduled activity.
2. Confirm the correct lesson or assessment is open.
3. Let the learner complete the session without manual difficulty changes.
4. Check the admin dashboard after submission.
5. Record only educational observations such as prerequisite gap, weak retention, concept misunderstanding, hesitation, rushing, or low confidence signal.
6. Export the required artifacts at the defined cadence.

## Issue Handling

Use the structured protocols in `content/pilot/aim_024_pilot_operations.json` for:

- login access
- lesson submission
- high frustration signal
- data quality gap

When a learner shows a high frustration signal, pause new difficulty exposure and use review or rest. Do not use clinical or diagnostic language.

## Data Closeout

At the end of day 13, the operator should have:

- pre-test and post-test scores
- session attempts
- adaptive recommendations
- recommendation outcomes
- admin daily snapshots
- documented absences or missing data gaps

The final pilot summary should focus on learning gain, lesson completion, recommendation effectiveness, retention movement, weakness reduction, and unresolved data quality gaps.
