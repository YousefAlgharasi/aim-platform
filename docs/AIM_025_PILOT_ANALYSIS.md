# AIM-025 Pilot Analysis

This closeout plan defines how to interpret the two-week AIM web pilot after AIM-023 readiness and AIM-024 operations are complete.

## Required Artifacts

- pre/post assessment scores
- session attempts
- adaptive recommendations
- admin daily snapshots
- operator notes

## Required Metrics

- learning gain
- lesson completion rate
- recommendation effectiveness
- mastery movement
- retention movement
- weakness reduction
- data completeness

## Interpretation Rules

Keep the report educational. Use terms such as prerequisite gap, weak retention, low confidence signal, hesitation, rushing, high frustration signal, and concept misunderstanding.

Do not use clinical language, diagnostic labels, or mental health claims. Do not use speed as a mastery or difficulty-increase signal.

## Decision

The closeout report should decide one of three next steps:

1. Repeat the five-learner pilot after content fixes.
2. Run a larger web pilot with the same architecture and stronger monitoring.
3. Move selected backend and React surfaces toward production hardening.

The decision must be based on evidence quality, algorithm safety, data completeness, and operator readiness.
